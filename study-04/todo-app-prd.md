# PRD: 개인용 할 일 관리 앱 (TodoFlow)

**문서 버전**: 1.0  
**작성일**: 2026-05-12  
**대상 독자**: 개발자(본인)

---

## 1. 개요 (Overview)

### 배경
매일 발생하는 10~20개의 할 일을 카테고리별로 관리하고, 진행 상황을 한눈에 파악할 수 있는 개인용 웹 앱이 필요하다.

### 목표
- 단일 HTML 파일(+ CSS/JS)로 구성된 가벼운 앱
- 별도 서버/DB 없이 브라우저 LocalStorage만으로 데이터 영속성 보장
- 모바일·데스크톱 양쪽에서 불편함 없이 사용 가능

---

## 2. 사용자 (Target User)

| 항목 | 내용 |
|------|------|
| 사용자 | 본인 1인 |
| 사용 기기 | PC 브라우저, 스마트폰 브라우저 |
| 사용 빈도 | 하루 수 회 (아침 계획 → 중간 체크 → 저녁 정리) |
| 일일 할 일 수 | 10~20개 |

---

## 3. 기능 요구사항 (Functional Requirements)

### 3-1. 할 일 관리 (CRUD)

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 추가 | 텍스트 입력 후 Enter 또는 버튼으로 할 일 추가. 카테고리 선택 필수 |
| F-02 | 수정 | 할 일 항목을 클릭하거나 수정 아이콘 탭으로 인라인 편집 가능 |
| F-03 | 삭제 | 삭제 아이콘 클릭 시 확인 없이 즉시 삭제 (실수 방지를 위해 2초간 Undo 토스트 제공) |
| F-04 | 완료 체크 | 체크박스 클릭으로 완료/미완료 토글. 완료 항목은 시각적으로 구분 (취소선 + 흐림) |

### 3-2. 카테고리 분류

| ID | 카테고리 | 색상 (예시) |
|----|----------|-------------|
| C-01 | 업무 | 파란 계열 |
| C-02 | 개인 | 초록 계열 |
| C-03 | 공부 | 주황 계열 |

- 카테고리 탭 클릭 시 해당 카테고리 항목만 필터링
- "전체" 탭으로 모든 항목 보기 가능

### 3-3. 진행률 보기

| ID | 기능 | 설명 |
|----|------|------|
| P-01 | 전체 진행률 | 상단에 완료/전체 개수와 퍼센트 표시 (프로그레스 바) |
| P-02 | 카테고리별 진행률 | 각 카테고리 탭에 완료/전체 카운트 뱃지 표시 |

### 3-4. 데이터 영속성

| ID | 기능 | 설명 |
|----|------|------|
| D-01 | 자동 저장 | 할 일 추가/수정/삭제/체크 시 즉시 LocalStorage에 저장 |
| D-02 | 자동 로드 | 페이지 로드(새로고침) 시 LocalStorage에서 데이터 복원 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| 기술 스택 | 순수 HTML + CSS + Vanilla JavaScript (프레임워크·라이브러리 없음) |
| 파일 구성 | `index.html` 1개 파일 (CSS·JS 인라인 또는 분리 파일 2개) |
| 반응형 | 320px(소형 모바일) ~ 1440px(데스크톱) 전 구간 정상 동작 |
| 브라우저 지원 | Chrome/Safari/Firefox 최신 버전 |
| 성능 | 20개 항목 기준 렌더링 100ms 이내 |
| 접근성 | 키보드 탐색 가능 (Tab 이동, Enter 확인, Escape 취소) |

## 5. UI/UX 레이아웃

```
┌─────────────────────────────────┐
│  TodoFlow          [날짜/요일]  │  ← 헤더
├─────────────────────────────────┤
│  ████████████░░░  65%  13/20   │  ← 전체 진행률 바
├─────────────────────────────────┤
│ [전체 20] [업무 8] [개인 6] [공부 6] │  ← 카테고리 탭 (필터)
├─────────────────────────────────┤
│  [카테고리 ▼] [ 할 일 입력... ] [+] │  ← 입력 영역
├─────────────────────────────────┤
│  ☑ 기획서 작성          업무  ✎ 🗑 │
│  ☐ 운동 30분            개인  ✎ 🗑 │
│  ☐ 알고리즘 문제 풀기   공부  ✎ 🗑 │
│  ...                            │  ← 할 일 목록
└─────────────────────────────────┘
```

**모바일 (≤ 480px)**
- 헤더 간소화
- 탭 가로 스크롤
- 수정/삭제 버튼 → 스와이프 또는 길게 누르기로 노출

## 6. 데이터 구조 (LocalStorage)

```javascript
// Key: "todoflow_tasks"
// Value: JSON 배열

[
  {
    "id": "1715500000000",        // Date.now() 문자열
    "text": "기획서 작성",
    "category": "업무",           // "업무" | "개인" | "공부"
    "completed": false,
    "createdAt": "2026-05-12T09:00:00.000Z"
  }
]
```

## 7. 화면 흐름 (User Flow)

```
앱 실행
  └→ LocalStorage 로드
       ├→ 데이터 있음 → 목록 렌더링
       └→ 데이터 없음 → 빈 상태 안내 메시지

할 일 추가
  └→ 카테고리 선택 + 텍스트 입력 + Enter/버튼
       ├→ 입력값 검증 (빈 문자열 차단)
       └→ 배열에 추가 → 저장 → 렌더링 갱신

완료 체크
  └→ 체크박스 클릭 → completed 토글 → 저장 → 진행률 갱신

삭제
  └→ 삭제 버튼 클릭 → 배열에서 제거 → 저장
       └→ 2초 Undo 토스트 표시 (시간 내 클릭 시 복원)
```
