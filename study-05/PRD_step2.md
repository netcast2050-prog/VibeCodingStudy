## 1. 개요 (Overview)

### 배경
Step 1에서 인식·편집된 재료 목록을 받아 실제로 만들 수 있는 레시피를 추천해야 한다.  
단순 레시피 나열이 아닌 재료 활용도가 높고 조리 난이도가 다양한 3가지 옵션을 제공한다.

### 목표
- 모델: `openai/gpt-oss-120b:free` (OpenRouter)
- 재료 기반 레시피 3개 생성 (난이도 다양화)
- 각 레시피: 이름·소요시간·난이도·재료 목록·단계별 조리법 포함
- 레시피를 Step 3(프로필)에 저장할 수 있는 진입점 제공

## 3. 기능 요구사항 (Functional Requirements)

### 3-1. 레시피 생성 요청

| ID | 기능 | 설명 |
|----|------|------|
| R-01 | 자동 요청 | Step 1에서 "레시피 생성하기" 버튼 클릭 시 재료 배열을 받아 즉시 API 요청 |
| R-02 | 로딩 상태 | 요청 중 스켈레톤 카드 3개 + "레시피를 만드는 중..." 메시지 표시 |
| R-03 | 재생성 | "다른 레시피 보기" 버튼으로 같은 재료로 새 레시피 3개 재요청 |
| R-04 | 재료 수정 후 재생성 | Step 1 재료 태그 편집 후 다시 생성 가능 |
| R-05 | 오류 처리 | API 오류·타임아웃 시 안내 메시지 + 재시도 버튼 |

**AI 프롬프트 설계**
```
시스템:
  당신은 한국 가정식 요리 전문가입니다.
  주어진 재료로 만들 수 있는 레시피 3가지를 추천해주세요.
  반드시 다음 JSON 형식으로만 응답하세요. 설명이나 마크다운 없이 JSON만 출력하세요.

  [
    {
      "name": "요리 이름",
      "time": "조리 시간(예: 20분)",
      "difficulty": "쉬움|보통|어려움",
      "servings": "인분 수(예: 2인분)",
      "usedIngredients": ["사용 재료1", "재료2"],
      "extraIngredients": ["추가 필요 재료 (없으면 빈 배열)"],
      "steps": ["1. 첫 번째 단계", "2. 두 번째 단계", "..."]
    }
  ]

  규칙:
  - 난이도는 쉬움/보통/어려움 중 하나씩 포함
  - 단계는 3~8개
  - 한국어로만 작성

사용자:
  재료: 달걀, 우유, 당근, 양파, 돼지고기
```

### 3-2. 레시피 카드 표시

| ID | 기능 | 설명 |
|----|------|------|
| C-01 | 카드 목록 | 생성된 레시피 3개를 세로 카드 형태로 나열 |
| C-02 | 카드 헤더 | 요리 이름 + 난이도 뱃지 + 소요시간 + 인분 수 |
| C-03 | 재료 섹션 | 사용 재료(초록 태그) / 추가 필요 재료(회색 태그) 구분 표시 |
| C-04 | 조리 단계 | 번호 목록으로 단계별 조리법 표시 |
| C-05 | 카드 펼치기 | 기본은 헤더+재료만 표시, 클릭 시 조리 단계 펼침(accordion) |

### 3-3. 레시피 저장

| ID | 기능 | 설명 |
|----|------|------|
| S-01 | 저장 버튼 | 각 카드에 "저장 ♡" 버튼, 클릭 시 Step 3 프로필에 저장 |
| S-02 | 저장 상태 표시 | 이미 저장된 레시피는 "저장됨 ♥" 로 표시 (localStorage 기준) |
| S-03 | 저장 토스트 | 저장 완료 시 2초 토스트 메시지 "레시피가 저장되었습니다" |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| 기술 스택 | 서버 프록시(`/api/recipe`) → OpenRouter GPT 호출, 프론트 Vanilla JS |
| 응답 형식 | 서버에서 JSON 파싱 후 구조화된 배열로 클라이언트 반환 |
| 파싱 안전성 | 모델 응답이 불완전한 JSON일 경우 재시도 1회 후 오류 처리 |
| 응답 시간 | 20초 이내 레시피 3개 생성 (초과 시 타임아웃 안내) |
| 반응형 | 모바일(≤ 480px)에서 카드 1열, 데스크톱에서 최대 3열 그리드 |

## 7. 화면 흐름 (User Flow)

```
Step 1 완료 → "레시피 생성하기" 클릭
  └→ Step 2 영역으로 스크롤
       └→ 스켈레톤 카드 3개 + 로딩 메시지 표시
            ├→ 성공 → 레시피 카드 3개 렌더링
            │         ├→ 카드 클릭 → 조리 단계 펼침
            │         └→ "저장 ♡" 클릭 → localStorage 저장 + 토스트
            └→ 실패 → 오류 메시지 + 재시도 버튼

"다른 레시피 보기" 클릭
  └→ 기존 카드 페이드아웃 → 스켈레톤 → 새 레시피 3개
```
