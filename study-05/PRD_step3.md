## 1. 개요 (Overview)

### 배경
매번 같은 재료로 같은 레시피를 생성하는 반복을 줄이고,  
마음에 드는 레시피를 모아두고 나중에 다시 꺼내 쓸 수 있는 개인 레시피 북이 필요하다.

### 목표
- 사용자 프로필(이름·인분 수·식이제한)을 설정하여 레시피 생성에 반영
- 저장된 레시피를 언제든 열람·재활용 가능
- 모든 데이터는 `localStorage`에 저장 (서버·DB 없음)

## 3. 기능 요구사항 (Functional Requirements)

### 3-1. 사용자 프로필 설정

| ID | 기능 | 설명 |
|----|------|------|
| P-01 | 프로필 초기 설정 | 앱 첫 실행 시 프로필 설정 모달 자동 표시 (localStorage에 프로필 없을 때) |
| P-02 | 닉네임 입력 | 1~10자 텍스트 입력 (예: "철수네 냉장고") |
| P-03 | 기본 인분 수 | 1·2·3·4인분 선택 → Step 2 레시피 생성 프롬프트에 자동 반영 |
| P-04 | 식이제한 선택 | 없음·채식·글루텐프리·유제품프리 중 복수 선택 가능 → 프롬프트에 반영 |
| P-05 | 프로필 수정 | 우상단 프로필 아이콘 클릭으로 언제든 재설정 가능 |
| P-06 | 프로필 저장 | "저장" 버튼 클릭 시 localStorage에 즉시 반영 |

### 3-2. 레시피 저장 관리

| ID | 기능 | 설명 |
|----|------|------|
| L-01 | 저장 목록 표시 | 저장된 레시피를 저장 최신순으로 나열 |
| L-02 | 레시피 카드 | Step 2와 동일한 카드 UI (이름·난이도·시간·인분 수·재료·조리 단계 아코디언) |
| L-03 | 삭제 | 카드의 삭제(🗑) 버튼 → 2초 Undo 토스트 제공 |
| L-04 | 빈 상태 | 저장된 레시피가 없을 때 안내 문구 + "레시피 만들러 가기" 버튼 |
| L-05 | 저장 개수 뱃지 | 네비게이션 또는 탭에 저장된 레시피 개수 표시 |

### 3-3. 레시피 필터·검색

| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 난이도 필터 | 전체·쉬움·보통·어려움 탭으로 필터링 |
| F-02 | 검색 | 요리 이름 또는 재료명으로 저장 목록 내 검색 |

### 3-4. 전체 앱 내비게이션

| ID | 기능 | 설명 |
|----|------|------|
| N-01 | 탭 내비게이션 | 하단(모바일) 또는 상단(PC) 탭: 📷 재료 인식 / 🍳 레시피 생성 / 📚 저장 레시피 |
| N-02 | 단계 상태 표시 | 현재 활성 탭 강조, 저장 탭에 저장 개수 뱃지 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| 데이터 저장 | 모든 프로필·레시피 데이터는 `localStorage`에 JSON으로 저장 |
| 프로필 반영 | `fridge_user_profile`을 Step 2 API 요청 시 `servings`·`dietary` 필드로 주입 |

| 브라우저 | Chrome/Safari/Firefox 최신 버전 |

## 7. 화면 흐름 (User Flow)

```
앱 첫 실행
  └→ localStorage에 프로필 없음
       └→ 프로필 설정 모달 자동 표시
            └→ 닉네임·인분수·식이제한 입력 → "저장하기"
                 └→ 모달 닫힘 → Step 1(재료 인식) 탭 진입

저장 탭 이동
  └→ 저장된 레시피 목록 표시
       ├→ 카드 클릭 → 조리 단계 아코디언 펼침
       ├→ 검색창 입력 → 실시간 필터링
       ├→ 난이도 탭 클릭 → 해당 난이도만 표시
       └→ 🗑 클릭 → 배열에서 제거 + 2초 Undo 토스트
            └→ Undo 클릭 → 복원

프로필 수정
  └→ 우상단 👤 아이콘 클릭 → 설정 모달 열림
       └→ 수정 후 "저장하기" → 반영 완료
```

## 8. 프로필 → 레시피 생성 반영 방법

Step 2 `/api/recipe` 요청 시 프로필 데이터를 자동 주입:

```javascript
// app.js (프론트엔드)
const profile = JSON.parse(localStorage.getItem('fridge_user_profile')) || {};

fetch('/api/recipe', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    ingredients: currentIngredients,
    servings: profile.servings ?? 2,
    dietary: profile.dietary ?? []
  })
});
```

서버 프롬프트에서 반영:
```
- 기본 인분 수: 2인분
- 식이제한: 채식 (육류·해산물 제외)
```
