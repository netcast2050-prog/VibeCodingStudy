## 1. 개요 (Overview)

### 배경
냉장고를 열었을 때 어떤 재료로 무엇을 만들 수 있는지 바로 알고 싶다.  
사진 한 장으로 재료를 자동 인식해 다음 단계(레시피 생성)로 넘기는 입력 레이어가 필요하다.

### 목표
- 이미지 업로드 → AI 재료 인식 → 재료 태그 확인·편집 → 레시피 생성으로 이동
- 모델: `google/gemma-4-31b-it:free` (OpenRouter)
- API 키는 서버 측에서만 사용 (브라우저에 노출 금지)

## 3. 기능 요구사항 (Functional Requirements)

### 3-1. 이미지 입력

| ID | 기능 | 설명 |
|----|------|------|
| I-01 | 파일 선택 | `<input type="file" accept="image/*">` 버튼으로 이미지 선택 |
| I-02 | 드래그&드롭 | PC 브라우저에서 이미지 파일을 업로드 존에 드래그하여 등록 |
| I-03 | 카메라 촬영 | 모바일에서 `capture="environment"` 속성으로 후면 카메라 직접 호출 |
| I-04 | 미리보기 | 선택된 이미지를 업로드 존 내에 섬네일로 즉시 표시 |
| I-05 | 파일 제한 | jpg·png·webp·heic 허용, 10MB 초과 시 오류 메시지 표시 |

### 3-2. AI 재료 인식

| ID | 기능 | 설명 |
|----|------|------|
| A-01 | 인식 요청 | "재료 인식" 버튼 클릭 시 이미지를 Base64로 인코딩 후 서버 프록시로 전송 |
| A-02 | 로딩 상태 | 요청 중 스피너 + "냉장고를 살펴보는 중..." 메시지 표시, 버튼 비활성화 |
| A-03 | 결과 파싱 | 모델 응답에서 재료명 목록을 추출하여 태그 배열로 변환 |
| A-04 | 오류 처리 | API 오류·타임아웃 시 안내 메시지 표시 + 재시도 버튼 제공 |

**AI 프롬프트 설계**
```
시스템: 냉장고나 식재료 이미지에서 재료를 식별하는 전문가입니다.
        재료명만 한국어로 콤마(,)로 구분하여 나열하세요. 설명은 하지 마세요.
        예시: 달걀, 우유, 당근, 양파, 돼지고기, 두부
사용자: [이미지]
```

### 3-3. 재료 태그 편집

| ID | 기능 | 설명 |
|----|------|------|
| E-01 | 태그 표시 | 인식된 재료를 삭제 가능한 칩(chip) 형태로 나열 |
| E-02 | 재료 삭제 | 태그의 × 버튼으로 불필요한 재료 제거 |
| E-03 | 재료 추가 | 입력 필드 + Enter로 재료 직접 추가 |
| E-04 | 재인식 | "다시 인식" 버튼으로 같은 이미지를 재요청 |

### 3-4. 다음 단계 이동

| ID | 기능 | 설명 |
|----|------|------|
| N-01 | 레시피 생성 버튼 | 재료가 1개 이상이면 활성화, 클릭 시 Step 2 영역으로 스크롤·전환 |
| N-02 | 재료 전달 | 확정된 재료 배열을 sessionStorage에 저장하여 Step 2에서 읽음 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| 기술 스택 | Node.js(Express) 서버 + 순수 HTML/CSS/Vanilla JS 프론트엔드 |
| API 보안 | OpenRouter API 키는 `.env`에만 보관, 서버 프록시(`/api/recognize`)를 통해서만 호출 |
| 이미지 전송 | 클라이언트→서버는 Base64, 서버→OpenRouter는 data URL(`data:image/jpeg;base64,...`) |
| 응답 시간 | 인식 요청 후 15초 이내 결과 반환 (초과 시 타임아웃 안내) |
| 반응형 | 모바일(≤ 480px) 기준 우선 설계, 데스크톱 확장 |
| 브라우저 | Chrome/Safari/Firefox 최신 버전 |

## 7. 화면 흐름 (User Flow)

```
앱 첫 진입
  └→ 업로드 존 표시 (빈 상태)

이미지 선택/촬영/드롭
  └→ 미리보기 표시
       └→ "재료 인식하기" 버튼 활성화

재료 인식 클릭
  └→ 로딩 스피너 표시
       ├→ 성공 → 재료 태그 렌더링
       │         └→ 태그 편집 (삭제/추가)
       │              └→ "레시피 생성하기" 클릭 → Step 2로 이동
       └→ 실패 → 오류 메시지 + 재시도 버튼
```

## 8. 서버 구조

```
study-05/
├── server.js           ← Express 서버 (API 프록시 포함)
├── public/
│   ├── index.html      ← 단일 페이지 (Step 1·2·3 섹션 포함)
│   ├── style.css
│   └── app.js
├── .env                ← OPENROUTER_API_Key (git 제외)
```

**서버 엔드포인트**

| 메서드 | 경로 | 역할 |
|--------|------|------|
| GET | `/` | `public/index.html` 서빙 |
| POST | `/api/recognize` | 이미지 → Gemma 모델 재료 인식 프록시 |
| POST | `/api/recipe` | 재료 → GPT 레시피 생성 프록시 (Step 2) |
