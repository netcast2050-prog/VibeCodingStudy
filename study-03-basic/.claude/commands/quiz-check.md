# 퀴즈 정답 정확성 검증

카테고리: $ARGUMENTS

## 검사 범위

- `$ARGUMENTS` 값이 **있으면** 해당 카테고리(`category` 필드가 일치하는 문제)만 검증합니다.
- `$ARGUMENTS` 값이 **없으면** `questions.js`의 모든 문제를 검증합니다.
- 카테고리 비교는 정확히 일치하는 문자열 기준입니다. 일치하는 카테고리가 없으면 사용 가능한 카테고리 목록을 안내해 주세요.

## 1. 데이터 무결성 검사

각 문제 객체에 대해 다음을 점검합니다.

1. **id 유일성**: 동일한 id가 중복되어 있지 않은가
2. **카테고리 표기 일관성**: 정해진 4개 카테고리(`한국사`, `세계지리`, `과학`, `예술과 문화`) 표기와 정확히 일치하는가 (공백 포함)
3. **difficulty 값**: `easy` / `medium` / `hard` 중 하나인가
4. **options 수**: 정확히 4개인가
5. **options 중복/공백**: 보기 중 중복되거나 공백만 있는 항목이 없는가
6. **correctAnswer 범위**: `0 ≤ correctAnswer < options.length` 인가
7. **explanation 존재**: 빈 문자열이 아니며 한 문장 이상인가
8. **불필요/누락 필드**: 스키마에 없는 필드나 누락된 필드가 있는가

## 2. 정답 정확성 검증

각 문제에 대해 다음을 수행합니다.

1. `correctAnswer` 인덱스가 가리키는 보기(`options[correctAnswer]`)가 질문에 대한 사실적·역사적·과학적 정답인지 확인합니다.
2. `explanation`의 내용이 정답을 뒷받침하는지, 정답과 모순되지 않는지 확인합니다.
3. 다른 보기들이 정답이 될 수 있는 해석이 있는지 점검합니다(다중 정답 위험).

검증 시 다음 원칙을 적용합니다.

- 변동성 정보(수도/순위/최신 기록 등)는 questions.js에 작성 시점·기준이 명시되어 있는지 확인합니다.
- 학설이 갈리는 정보는 주류 학설을 기준으로 판단하고, 해설에 다른 학설이 부기되어 있는지 확인합니다.
- **자신 없는 항목은 추측하지 말고 "재확인 필요"로 분류합니다.** 어느 출처를 확인해야 할지 명시해 주세요.

## 3. CLAUDE.md 가이드라인 점검

각 문제에 대해 다음 4개 항목을 통과하는지 점검합니다.

1. 정답이 하나뿐인가? (다른 해석 가능 시 조건 명시)
2. 최상급 표현(`가장`, `최초`, `최대` 등)에 측정 기준이 있는가?
3. 시간과 범위가 명확한가? (변동 정보의 시점·지리적·분류적 범위)
4. 교차 검증이 필요한 논란성 정보인가?

## 출력 형식

```
# 퀴즈 정답 정확성 검증

## 검사 범위
- 대상: 전체 / 카테고리(__)
- 검증 문제 수: N개

## 1. 데이터 무결성
| 항목 | 결과 | 상세 |
|------|------|------|
| id 중복 | ✅ / ⚠️ | id: __ |
| 카테고리 표기 | ✅ / ⚠️ | id: __ ("__" 표기 이상) |
| difficulty 값 | ✅ / ⚠️ | id: __ |
| options 수 | ✅ / ⚠️ | id: __ (보기 N개) |
| options 중복 | ✅ / ⚠️ | id: __ |
| correctAnswer 범위 | ✅ / ⚠️ | id: __ |
| explanation 누락 | ✅ / ⚠️ | id: __ |

## 2. 정답 정확성 (이슈만 표시)
| id | 카테고리 | 질문(축약) | 등록 정답 | 검토 의견 | 위험도 |
|----|---------|----------|----------|----------|--------|
| 12 | 과학    | __       | options[2] | 사실 오류 가능 — options[1]이 정답일 수 있음 | 상 |

## 3. 가이드라인 위반
| id | 조항 | 내용 | 권장 수정 |
|----|------|------|----------|
| 7  | 최상급 기준 누락 | "가장 긴 강" — 길이 기준 명시 필요 | "길이 기준, 한반도에서 가장 긴 강은?" |

## 요약
- 무결성 위반: N건
- 정답 의심: N건 (상 X / 중 Y / 하 Z)
- 가이드라인 위반: N건
- 재확인 필요: N건

## 우선 수정 권장 (위험도 '상' 상위 5건)
1. id __: 사유 / 권장 조치
2. id __: ...
```

## 마무리

- 정답에 의심이 가지만 즉시 단정할 수 없는 문항은 **"재확인 필요"** 상태로 표시하고, 어떤 출처(예: 표준국어대사전, 위키백과의 특정 항목, 교육부 검정 교과서 등)를 확인해야 할지 함께 적습니다.
- 본 명령은 **`questions.js`를 수정하지 않습니다.** 보고만 하고, 사용자가 수정 여부를 결정하게 합니다.
- 검사할 파일이 없거나 해당 카테고리의 문제가 없으면 그 사실을 명확히 알려 주세요.
