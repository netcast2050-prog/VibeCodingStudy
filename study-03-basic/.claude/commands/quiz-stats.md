# 퀴즈 게임 통계 분석

모드: $ARGUMENTS

## 분석 범위

- `questions.js` 전체 문제 풀을 대상으로 통계를 산출합니다.
- `$ARGUMENTS`가 `brief`이면 한눈에 요약만, `full` 또는 비어 있으면 상세 통계를 출력합니다.
- 본 명령은 **파일을 수정하지 않습니다.** 읽기·집계·보고만 수행합니다.

> **참고**: 게임 결과 통계(점수 추이, 정답률 등)는 사용자 브라우저의 `localStorage` (`quizApp.history`) 에 저장되므로 이 명령으로는 직접 조회할 수 없습니다. 사용자가 그 JSON을 메시지에 첨부했을 때만 별도로 분석합니다(섹션 8 참조).

## 산출 항목

### 1. 전체 통계
- 총 문제 수
- 카테고리 수와 목록
- 사용 중인 난이도 종류와 개수

### 2. 카테고리별 분포
- 카테고리별 문제 수와 비율
- 카테고리별 난이도 분포 (easy / medium / hard)
- 카테고리별 정답 인덱스 분포 (0 / 1 / 2 / 3)
- 균형 평가: 특정 카테고리가 전체의 35% 이상이거나 15% 이하면 경고

### 3. 난이도 분포
- 전체 난이도 비율
- 권장 분포(`easy` 30~40% · `medium` 35~45% · `hard` 20~30%)와의 차이
- 카테고리별로 난이도 누락(0개) 항목

### 4. 정답 인덱스 분포
- 전체 `correctAnswer` 분포 (0 / 1 / 2 / 3 비율)
- 편중 여부: 한 인덱스가 35% 이상이면 편중 경고
- 카테고리별 편중 여부

### 5. 텍스트 길이 통계
- 질문 길이: 평균 / 최소 / 최대
- 보기 평균 길이
- 해설 평균 길이
- 이상치: 비정상적으로 짧은(<10자) 또는 긴(>200자) 항목 목록

### 6. 중복·유사 위험
- 동일 키워드를 공유하는 질문 페어 (예: 같은 인물·사건·연도)
- 거의 동일한 보기 세트를 공유하는 문제

### 7. 게임 모드 적합성
`script.js`의 `gameModes` 정의 기준으로 판정합니다.

- `mode: full` (40문제 추출): 카테고리별 균등 추출이 가능한가 (각 카테고리 ≥ 10문제 권장)
- `mode: category` (10문제): 각 카테고리에 10문제 이상 있는가
- `mode: speed` (20문제): 충분한 풀이 있는가

부족한 카테고리·난이도가 있으면 명시합니다.

### 8. (선택) 첨부된 history JSON 분석
사용자가 메시지에 `quizApp.history` 형태의 JSON을 직접 붙여넣은 경우에만 다음을 추가로 산출합니다.

- 총 플레이 횟수, 평균 점수, 최고 점수
- 모드별·카테고리별 평균 정답률
- 최근 10회 점수 추이
- 신기록 갱신 이력

JSON이 없으면 이 섹션은 출력하지 않습니다.

## 출력 형식

```
# 퀴즈 풀 통계 (YYYY-MM-DD 기준)

## 한눈에
- 총 문제: N개 · 카테고리 K개 · 난이도 D종
- 균형 점수: ★★★★☆ (4/5) — 카테고리 분포 양호 / 난이도 hard 부족 등

## 카테고리별 분포
| 카테고리 | 문제 수 | 비율 | easy | medium | hard | 정답 0/1/2/3 |
|---------|--------|------|------|--------|------|-------------|
| 한국사   | 13     | 32%  | 3    | 7      | 3    | 4/3/3/3     |
| 세계지리 | ...    | ...  | ...  | ...    | ...  | ...         |
| 과학    | ...    | ...  | ...  | ...    | ...  | ...         |
| 예술과 문화 | ... | ...  | ...  | ...    | ...  | ...         |

## 난이도 분포 (전체)
- easy: N (X%)
- medium: N (X%)
- hard: N (X%)
- 권장 대비: ___ 조정 필요

## 정답 인덱스 분포
- 0: N (X%) · 1: N (X%) · 2: N (X%) · 3: N (X%)
- 편중 경고: ___ (있는 경우만)

## 텍스트 길이
- 질문: 평균 __자 (최소/최대 __/__자)
- 보기: 평균 __자
- 해설: 평균 __자
- 이상치: id __ (질문 __자), id __ (해설 __자), ...

## 중복·유사 의심
| id A | id B | 유사 사유 |
|------|------|----------|
| 5    | 22   | 동일 인물(__)에 대한 질문 |

## 게임 모드 적합성
- full(40): ✅ 충분 / ⚠️ N개 부족
- category(10): ✅ 모든 카테고리 가능 / ⚠️ ___ 카테고리 N개 부족
- speed(20): ✅ / ⚠️

## 권장 조치
1. ___ 카테고리에 hard 난이도 N개 추가 권장 (현재 0개)
2. ___ 카테고리 정답 0번 편중(50%) — 다음 추가 시 1~3번에 배치
3. id __ ↔ id __ 유사 — 통합 또는 차별화 검토
```

## 마무리

- 통계 결과만 보고하며 `questions.js`나 다른 파일을 수정하지 않습니다.
- 권장 조치는 우선순위(영향도)가 높은 순서로 정렬해 주세요.
- 분석 대상이 비어 있거나 파싱이 불가능하면 그 사실을 분명히 알려 주세요.
