# 퀴즈 문제 추가

새 퀴즈 문제를 생성하여 `questions.js`에 추가합니다.

- **카테고리**: $1
- **난이도**: $2

## 입력 검증

다음 조건을 먼저 확인합니다. 충족하지 않으면 진행하지 말고 사용자에게 사용 가능한 값을 안내해 주세요.

1. `$1`이 비어 있지 않고, 기존 카테고리 중 하나여야 합니다.
   - 허용: `한국사`, `세계지리`, `과학`, `예술과 문화`
   - 새 카테고리를 추가하려는 경우라면 사용자에게 의도를 한 번 확인합니다.
2. `$2`가 비어 있지 않고, 다음 중 하나여야 합니다.
   - 허용: `easy`, `medium`, `hard`
   - `하/중/상` 등 한글 표기가 들어오면 `easy/medium/hard`로 정규화합니다.
3. 두 값 중 하나라도 누락된 경우 어느 인자가 빠졌는지 명시하고 종료합니다.

## 수행 작업

### 1. 기존 문제 분석
- `questions.js`(또는 동등한 퀴즈 파일)를 읽어 다음을 파악합니다.
  - 사용 중인 가장 큰 `id` 값 → 신규 `id`는 `max(id) + 1`
  - 해당 카테고리에 이미 등록된 질문 목록 → **중복 주제 방지**
  - 같은 카테고리·난이도 문제들의 보기 정답(correctAnswer) 분포 → **정답 편중 회피**

### 2. 문제 설계 (CLAUDE.md 가이드라인 준수)

작성 전에 다음 4가지를 모두 충족하는지 점검합니다. 하나라도 못 지키면 문제를 다시 설계하세요.

1. **정답이 하나뿐인가?** 다른 해석 가능성이 있다면 조건(예: 면적 기준, 2024년 기준)을 질문에 명시합니다.
2. **최상급 표현에 측정 기준이 있는가?** `가장`, `최대`, `최초`, `유일한` 등의 표현을 쓸 때 반드시 기준(면적/인구/질량/연도/범위 등)을 함께 적습니다.
3. **시간·범위가 명확한가?** 변할 수 있는 정보(수도, 1위, 최신 기록 등)는 시점을 명시하고, 지리적·분류적 범위를 한정합니다.
4. **교차 검증이 끝났는가?** 논란이 있는 정보(예: 가장 긴 강)는 주류 학설을 따르고, 해설에 다른 학설을 부기합니다.

추가 설계 원칙:
- 보기 4개는 모두 **그럴듯한 오답**으로 구성하고, 같은 카테고리에서 익숙한 단어를 활용합니다.
- 정답 인덱스(`correctAnswer`)는 같은 카테고리·난이도 그룹의 분포를 보정하는 방향으로 선택합니다(특히 4번/index 3이 부족하면 우선 배치).
- 해설은 한두 문장으로 정답의 근거를 적고, 필요한 경우 시점·기준·논란을 짧게 부기합니다.

### 3. 파일에 삽입

- 위치는 같은 카테고리 블록의 마지막 항목 다음입니다. 마지막 항목과의 쉼표·중괄호 처리에 주의합니다.
- 같은 카테고리 블록을 찾기 어려우면 배열 끝에 추가하되, 닫는 `]` 직전 항목의 쉼표 누락을 확인합니다.
- 삽입 후 JS 객체 리터럴 문법이 깨지지 않았는지 다시 읽어 확인합니다.

객체 형식은 기존 문제와 동일해야 합니다.

```js
{
    id: <max+1>,
    category: "$1",
    difficulty: "$2",
    question: "...",
    options: ["...", "...", "...", "..."],
    correctAnswer: <0-3>,
    explanation: "..."
}
```

### 4. 자체 검증

추가 직후 다음을 자체 점검하고 결과를 보고합니다.

- 신규 `id`가 기존 값과 충돌하지 않는가
- `options`가 정확히 4개이고 중복이 없는가
- `correctAnswer`가 0~3 범위이며 `options[correctAnswer]`가 실제 정답인가
- 같은 카테고리에 동일 주제의 기존 문제가 없는가
- CLAUDE.md 가이드라인 4개 항목을 모두 통과하는가

## 출력 형식

```
# 추가된 퀴즈

- id: <번호>
- 카테고리: $1
- 난이도: $2
- 질문: <질문 본문>
- 보기:
  1. <보기1>
  2. <보기2>
  3. <보기3>
  4. <보기4>
- 정답: <번호> (<보기 본문>)
- 해설: <해설 본문>

## 검증 결과
- 정답 유일성: ✅ / ⚠️ (상세)
- 최상급 기준 명시: ✅ / ⚠️ / 해당 없음
- 시간·범위 명시: ✅ / ⚠️ / 해당 없음
- 교차 검증: ✅ / ⚠️ (참조 또는 보완 필요 사항)

## 분포 영향
- 같은 카테고리·난이도 정답 분포 변화: 1번 a→a' / 2번 b→b' / 3번 c→c' / 4번 d→d'

## 삽입 위치
- 파일: questions.js
- 라인: 약 L<번호> (카테고리 블록 끝)
```

## 실패/중단 처리

- 입력 검증 실패: 어떤 인자가 잘못됐는지와 허용 값을 알려주고 작업을 중단합니다.
- 퀴즈 파일을 찾지 못한 경우: 탐색한 경로와 결과를 보고하고 사용자에게 파일 위치를 묻습니다.
- 가이드라인을 만족하는 문제를 만들 수 없다고 판단되면, 추가하지 말고 그 이유를 설명합니다.
